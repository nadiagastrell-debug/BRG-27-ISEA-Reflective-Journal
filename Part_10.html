MySQL Database Server

For my additional service, I chose MySQL because databases power most websites and applications.

Step 1: Install MySQL

 sudo apt update
	      sudo apt install mysql-server
	      sudo mysql_secure_installation

The secure installation wizard lets me set root passworkd, remove test accounts, and disable remote root login.

Step 2: Login to MySQL

sudo mysql -u root -p

This logs me in as root user and prompts for a password.

Step 3: Create Databases

CREATE DATABASE testdb;
	     SHOW DATABASES;
	     USE testdb;

These commands create a database called testdb, list all databases and select testdb to work with.

Step 4: Create Tables

CREATE TABLE users (
	     id INT AUTO_INCREMENT PRIMARY KEY,
	     username VARCHAR(50) NOT NULL,
	     email VARCHAR(100) NOT NULL,
	    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
 	 );

	    SHOW TABLES;
	    DESCRIBE users;

This creates a users with auto-incrementing ID, username, email and timestamp columns.

Step 5: Inserting Data

INSERT INTO users (username, email) VALUES ('testuser', 'testuser@example.com');
	     INSERT INTO users (username, email) VALUES ('john', 'john@example.com');
         INSERT INTO users (username, email) VALUES ('alice', 'alice'@'example.com');

Here I am adding three test users to the database to show how it works.

Step 6: Querying Data

SELECT * FROM users;

This command shows all users in the database testdb with all columns.

Step 7: Creating Application Users

CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'passworkd';
	     GRANT ALL PRIVILEGES ON testdb.* TO 'appuser'@'localhost';
	     FLUSH PRIVILEGES;
	     SELECT user, host FROM mysql.user  WHERE user = 'appuser';

These prompts create a dedicated user with limited permissions for security. Applications should never use the root account. The last line will show only specific columns for a specified user.

What I Learned

MySQL stores organized data in tables. Every website that has users, posts or even products uses databases like this. The SQL commands (CREATE, INSERT, SELECT) are how applications talk to databases to save and retrieve information.
